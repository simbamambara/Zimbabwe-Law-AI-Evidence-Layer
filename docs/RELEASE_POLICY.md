# Release Policy

1. Full data remains private by default.
2. Every release has an immutable tag, changelog, manifest, and validation report.
3. Data changes require source evidence and authorised human approval.
4. Privacy review precedes public release.
5. A correction or suppression creates a new release; prior evidence remains access-controlled.
6. Release notes disclose known limitations and unresolved manual-review items.
7. Public-minimal data is released only after formal sign-off.
