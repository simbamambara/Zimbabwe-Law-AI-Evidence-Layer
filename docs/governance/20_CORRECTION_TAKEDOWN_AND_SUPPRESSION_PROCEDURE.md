# Correction, Takedown and Suppression Procedure

1. Log the request and acknowledge receipt.
2. Verify the requester and record identity.
3. Preserve the disputed version and source evidence.
4. Assess accuracy, official source status, public interest, legal obligations, and harm.
5. Correct derived metadata, suppress public access, or deny with reasons as appropriate.
6. Issue a new release and update manifests; never silently rewrite history.
7. Notify downstream recipients where feasible.
8. Record approval and closure.

Use `docs/registers/correction_takedown_register.csv`.
