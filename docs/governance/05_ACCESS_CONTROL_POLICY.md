# Access Control Policy

Full-text and source-block data is restricted. Access is least-privilege, role-based, approved by the data owner, and reviewed quarterly.

Required controls: unique accounts, multi-factor authentication where available, no shared credentials, encrypted devices, prompt revocation, access logging, private repositories, branch protection, secret scanning, and controlled exports.

Use `docs/registers/access_review_register.csv` for periodic review. Public-minimal data must be stored separately from restricted data.
